# 广告知识库

包含收到的广告传单、名片信息，整理成结构化 JSON，用于搭建 AI 知识库。

通过统一的数据格式与 `"key"` 关联机制，将公司与个人信息互相链接，便于查询、检索与后续的程序化调用。

## 数据格式

核心数据存放于 [广告信息.json](广告信息调用/广告信息/广告信息.json)，包含两个顶层数组：

```json
{
	"公司信息": [ ... ],
	"个人信息": [ ... ]
}
```

- 每个对象以 `"key"` 作为唯一标识，用于跨对象关联（如公司的 `关联个人`、个人的 `关联企业`）。
- 公司 key 形如 `china_{统一社会信用代码}` 或 `{国籍}_{本地税号}`；个人 key 形如 `{姓拼音}_{名拼音}_{电话号码}`。
- 序列化约定：`indent="\t"`、`ensure_ascii=False`（中文正常显示）。

完整的字段定义、命名约束与录入流程见 [广告信息/README.md](广告信息调用/广告信息/README.md) 与 [格式要求.md](广告信息调用/广告信息/AI辅助文件/格式要求.md)。

## 解析工具

[utils/json解析器.py](广告信息调用/utils/json解析器.py) 提供 `对象数组解析器`，把形如 `[{"key": ...}, ...]` 的对象数组转换成以 key 为索引的结构：

| 成员 | 说明 |
|------|------|
| `len(解析器)` | 对象个数 |
| `解析器[key]` | 按 key 取对象（已去除 `key` 字段） |
| `key in 解析器` | 是否存在该 key |
| `共通属性` | 所有对象都具备的属性集合 |
| `所有属性` | 出现过的全部属性集合 |
| `非共通属性` | `所有属性 - 共通属性` |
| `提取属性值(属性名)` | `{ key: 该属性值 }` |
| `keys() / values() / items()` | 字典式遍历 |

用法示例见 [辅助测试项目/json数据解析/调用.py](辅助测试项目/json数据解析/调用.py)。

## 使用方法

```bash
# 运行解析器测试样例
cd 辅助测试项目/json数据解析
python 调用.py

# 解析广告信息数据
cd 广告信息调用
python 调用.py
```

> 当前 [广告信息调用/调用.py](广告信息调用/调用.py) 仍为待填充的入口骨架，可在此基础上加载 `广告信息.json` 并用 `对象数组解析器` 进行查询。

## 维护数据

录入新公司 / 个人或做一致性检查时，可参考 `AI辅助文件/` 中的提示词：

- **新项目提示词** —— 把广告或名片归纳成符合格式的 JSON 条目。
- **一致性检查提示词** —— 检查 JSON 语法、同义不同名的 key、类型不一致等问题。

> `AI辅助文件/` 已在 `.gitignore` 中忽略，不纳入版本管理。
