## 格式要求

### 一、整体结构

文件包含两个顶层数组：

```json
{
	"公司信息": [ ... ],
	"个人信息": [ ... ]
}
```

- 每个数组内的对象使用 **`"key"`** 作为唯一标识，方便跨对象关联。
- **不添加** 未出现在原始广告/名片中的字段，也不留空字段。

---

### 二、公司信息对象（`公司信息` 数组内）

#### 2.1 必填核心字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `key` | string | 唯一标识：`"china_{统一社会信用代码}"` 或 `"{国籍小写}_{该国唯一编码}"` | `"china_91440400MA4WAGEK48"`<br>`"germany_DE128249150"` |
| `统一社会信用代码` | string | 中国企业的 18 位代码；外企可省略或用当地税号等 | `"91440400MA4WAGEK48"` |
| `公司名称` | string | 全称，中文或外文 | `"珠海光恒科技有限公司"` |

#### 2.2 地址表示（`地点` 字段）

- 类型：对象数组
- 每个对象包含：
  - `"key"`：地址类型，通常为 `"默认地址"`、`"总部"`、`"注册地址"`、`"北京子公司"` 等
  - `"value"`：详细地址字符串

```json
"地点": [
	{ "key": "默认地址", "value": "广东省珠海市香洲区永南路388号..." },
	{ "key": "总部", "value": "珠海市香洲区力合光电产业园内" }
]
```

#### 2.3 产品表示（`产品` 字段）

- 类型：对象数组（**即使只有一个产品也使用数组**）
- 每个产品对象 **必须包含** `"key"`（自定义，保证在数组内唯一），其余字段按实际信息添加（如 `"名称"`、`"型号"`、`"特点"`、`"分辨率"`、`"接口"` 等）

```json
"产品": [
	{
		"key": "EXO",
		"名称": "EXO系列",
		"特点": [ "基于索尼...", "精密机加工..." ],
		"型号": [ ... ]
	}
]
```

- 若产品信息非常简略，可直接用字符串数组：

```json
"产品": [ "红外探测器", "热成像机芯模组" ]
```

#### 2.4 关联人员（`关联个人` 字段）

- 类型：字符串数组，元素为 **个人对象中的 `key`**
- 用于链接公司与人

```json
"关联个人": [ "Ding_13411397528", "Sonia_Sun_15813880878" ]
```

#### 2.5 其他常见可选字段（按需添加）

| 字段 | 类型 | 示例 |
|------|------|------|
| `母公司` | string | 父公司的 `key` |
| `子公司` | string[] | 子公司 `key` 数组 |
| `员工规模` | [number, number] | `[500,600]` 表示区间 |
| `研发人员占比` | number | `0.35` |
| `工作时间` | object | `{"周":"五天八小时双休","日":["8:30","17:30"],"法定节假日":true}` |
| `招聘关注` | object | 包含 `福利`、`岗位` 等（见原文件示例） |
| `官网` | string | URL |
| `电话` / `邮箱` | string / string[] | `"400-998-9038"` 或 `["0755-84687805"]` |
| `附属文件` | string[] | 相对路径，如 `"files/xxx/公众号.jpg"` |
| `其他关键词` | string[] | 补充说明 |

---

### 三、个人信息对象（`个人信息` 数组内）

#### 3.1 必填核心字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `key` | string | `"{姓拼音}_{名拼音}_{电话号码}"` 或 `"{英文名}_{电话号码}"`，无空格，下划线分隔 | `"Sonia_Sun_15813880878"` |
| `姓名` | string | 中文全名，或外文常用名 | `"孙霜"` |
| `电话` | string | 完整手机号（不加空格） | `"15813880878"` |

#### 3.2 常用可选字段

| 字段 | 类型 | 示例 |
|------|------|------|
| `微信` | string | 通常同手机号 |
| `QQ` | string | `"105785955"` |
| `邮箱` | string | `"barry@levtop.cn"` |
| `职位` / `职务` | string | `"人事"`、`"Sales Director"` |
| `部门` | string | `"业务部"` |
| `别名` | string[] | 曾用名或英文名 |
| `关联企业` | string[] | 所任职公司的 `key` 数组 |

```json
"关联企业": [ "china_91440300311898815T" ]
```

#### 3.3 特殊字段（视情况）

- `门店地址`：仅当个人名片上明确给出时添加
- `Skype` 等即时通讯账号

### 四、关键风格与约束

1. **唯一标识（key）生成规则**
   - 公司：`"china_{统一社会信用代码}"`（国内） / `"{国籍}_{本地唯一编码}"`（国外）
   - 个人：`"{姓拼音}_{名拼音}_{电话号码}"` 或 `"{英文名}_{电话号码}"`（英文名优先采用名片上的写法）

2. **数组 vs 对象**
   - 凡是可能含多个值的字段，一律用数组（即使当前只有一个值）。例如：`"电话": ["0755-84687805"]`，`"产品": [ ... ]`
   - 但 `地点` 内的每个地址元素本身是对象，所以用对象数组。

3. **嵌套层级**
   - 产品规格、技能要求等可嵌套对象或数组，保持可读性。
   - 禁止过深嵌套（一般不超过 4 层）。
   - 若出现数组内的成员为对象的时候，这些对象需要添加`"key"`合适的字段，其内容在一定程度保证可读性与非重复性的条件下自由发挥

4. **字段命名**
   - 使用中文键名（如 `"公司名称"`、`"统一社会信用代码"`），保持全文件一致。
   - 仅英文专有名词或缩写保留大写（如 `"key"`、`"etc."`）。

5. **信息保真**
   - 原始广告/名片中没有的信息 **不添加**。
   - 原始信息中的错别字或格式问题 **保留原样**（例如 `"etc."` 直接保留）。
   - 附件路径、URL 等按原文填写。

6. **子公司/母公司关联**
   - 使用 `"母公司"` 或 `"子公司"` 字段，值为对方公司的 `key`。
   - 避免循环引用，但允许单向关联。

7. **文件外链**
   - `"附属文件"` 数组存储相对路径，路径前缀如 `"files/公司名/文件名"`，一般有二维码的需要添加该字段。

---

### 五、添加新条目的流程建议

1. 确定要添加的是 **公司** 还是 **个人**。
2. 收集原始文本信息，去重、归类。
3. 生成唯一 `key`（按规则）。
4. 按上述表格填充字段：
   - 先填必填核心字段
   - 再填可选常用字段（地点、产品、电话等）
   - 最后填特殊字段（招聘信息、技术参数等）
5. 检查所有数组字段是否使用 `[]`，即使是空数组或单元素数组。
6. 如果存在关联（人员在公司工作，或子公司关系），在两端各自填入对方的 `key`。
7. 保持缩进：使用 **制表符（`\t`）** 缩进，确保 `ensure_ascii=False`（即中文正常显示）。
8. 请不要生成完整的 json 文件，请用以下格式提示相关人员添加条目的方法
将以下内容添加到 `公司信息`
```json
...
```
将以下内容添加到 `个人信息`
```json

```

	"产品": [ "智能相机", "视觉软件" ],

		{
